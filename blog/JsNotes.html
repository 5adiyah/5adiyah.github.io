Notes from Codecademy Lessons - I’ve extracted them here for easier reference. All credits for the following notes are due to Codecademy. Visit their site to practice implementing these lessons.

What can we use JavaScript for?

  • Make websites respond to user interaction
  • Build apps and games (e.g. blackjack)
  • Access information on the Internet (e.g. find out the top trending words on Twitter by topic)
  • Organize and present data (e.g. automate spreadsheet work; data visualization)

Data Types

Data comes in various types:

  1. Numbers are quantities, just like you’re used to. You can do math with them.
  2. Strings are sequences of characters, like the letters a-z, spaces, and even numbers. Strings are extremely useful as labels, names, and content for your programs. These are all strings: “Ryan” “4” and “What is your name?” To write a string, surround words with quotes.
  3. **Boolean A boolean is either true or false. 23 > 10 is true **

Using console.log

  • console.log() will take whatever is inside the parentheses and log it to the console below your code
  • This is commonly called printing out

Comparisons

List of comparison operators:

  • Greater than >
  • Less than >
  • Less than or equal to <=
  • Greater than or equal to >=
  • Equal to ==
  • Not equal to !==

Manipulating numbers & strings

  • Comparisons (e.g. > , <=)
  • Modulo (e.g. %)
  • String length (e.g. "Emily".length;)
  • Substrings (e.g. "hi".substring(0, 1);)

If Statements

An if statement is made up of the if keyword, a condition, and a pair of curly braces { }. If the answer to the condition is yes, the code inside the curly braces will run.

Variables

To do more complex coding, we need a way to ‘save’ the values from our coding. We do this by defining a variable with a specific, case-sensitive name. Once you create (or declare) a variable as having a particular name, you can then call up that value by typing the variable name.

var varName = data;

Function Syntax

  • **var declares a variable named functionName**
  • The keyword function tells the computer that
  • **functionName** is a function and not something else.
  • **Parameters go in the parenthesis after function**
  • The code block is the reusable code that is between the {}
  • Each line of code inside { } must end with a semi-colon.
  • The entire function must end with a semi-colon.
  • To use the function, we call the function by typing the function’s name and putting a parameter value inside parenthesis after it.

Global vs Local Variables

Scope can be local or global:

Global Variables

  • Variables defined outside a function are accessible anywhere once they have been declared. They are called global variables and their scope is global.
  • The variable globalVar can be accessed anywhere, even inside the function foo.h

Local Variables

  • Variables defined inside a function are local variables.
  • They cannot be accessed outside of that function. Their scope is local.
  • The variable localVar only exists inside the function bar. Trying to print localVar outside the function gives a error.
